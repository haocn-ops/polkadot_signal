# Polkadot Signal - 完整架构文档

## 项目概述

基于Polkadot/Substrate的去中心化端到端加密消息应用，复用Signal协议实现隐私保护通讯。

## 模块说明

### 1. Pallets (Substrate Runtime模块)

| Pallet | 功能 | 主要Extrinsics |
|--------|------|---------------|
| `signal-keys` | Signal密钥注册 | `register_identity`, `add_one_time_prekeys`, `remove_identity` |
| `signal-groups` | 群组管理 | `create_group`, `invite_member`, `accept_invite`, `remove_member` |
| `message-queue` | 离线消息队列 | `notify_message`, `mark_read`, `confirm_delivery` |

### 2. 协议层 (Signal Protocol)

| 模块 | 功能 |
|------|------|
| `keys.rs` | Ed25519/X25519密钥生成、签名验证 |
| `x3dh.rs` | X3DH密钥协商协议 |
| `double_ratchet.rs` | 双棘轮算法（前向保密） |
| `session.rs` | 端到端加密会话管理 |
| `message.rs` | 消息封装格式 |

### 3. P2P网络层

| 组件 | 功能 |
|------|------|
| Gossipsub | 消息广播/订阅 |
| Kademlia | DHT节点发现 |
| Identify | 节点身份识别 |
| Ping | 连接保活 |

### 4. 存储层

| 模块 | 功能 |
|------|------|
| IpfsClient | IPFS API交互 |
| MessageStore | 加密消息存储 |
| OfflineQueue | 离线消息队列 |

## 消息流程

### 发送消息流程

```
Alice发送消息给Bob:
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  Alice  │────▶│ Encrypt │────▶│   P2P   │────▶│  IPFS   │────▶│  Chain  │
│  Client │     │ (Signal)│     │  Send   │     │ Store   │     │ Notify  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘
     │               │               │               │               │
     │               │               │               │               │
     ▼               ▼               ▼               ▼               ▼
  输入消息      X3DH+Ratchet     libp2p直连     加密存储CID    通知Bob
               加密消息         或广播         (可选)
```

### 接收消息流程

```
Bob接收消息:
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│   Bob   │────▶│  Chain  │────▶│  IPFS   │────▶│ Decrypt │────▶│  Show   │
│  Client │     │ Query   │     │ Fetch   │     │ (Signal)│     │ Message │
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘
     │               │               │               │               │
     │               │               │               │               │
     ▼               ▼               ▼               ▼               ▼
  上线检测      获取CID        下载加密消息    Double Ratchet   显示明文
               通知          (如离线)       解密
```

## 加密流程详解

### X3DH密钥交换

```
Alice (发起方)                              Bob (接收方)
     │                                           │
     │  1. 获取Bob的密钥包                        │
     │     - Identity Key (IK_B)                 │
     │     - Signed Prekey (SPK_B)               │
     │     - One-time Prekey (OPK_B)             │
     │                                           │
     │  2. 生成临时密钥                           │
     │     - Ephemeral Key (EK_A)                │
     │                                           │
     │  3. 计算DH                                │
     │     DH1 = DH(IK_A, SPK_B)                 │
     │     DH2 = DH(EK_A, IK_B)                  │
     │     DH3 = DH(EK_A, SPK_B)                 │
     │     DH4 = DH(EK_A, OPK_B)                 │
     │                                           │
     │  4. 派生根密钥                             │
     │     SK = KDF(DH1 || DH2 || DH3 || DH4)    │
     │                                           │
     └───────────────────────────────────────────▶
```

### Double Ratchet消息加密

```
发送方                                      接收方
  │                                           │
  │  每条消息:                                 │
  │  1. 从发送链派生消息密钥                    │
  │  2. 加密消息                               │
  │  3. 发送 (密文 + 棘轮公钥)                  │
  │                                           │
  │  收到新棘轮公钥时:                          │
  │  1. 执行DH棘轮                             │
  │  2. 更新发送链                             │
  │  3. 创建新接收链                           │
  │                                           │
```

## 构建和运行

### 前置要求

- Rust 1.70+
- Node.js 18+
- IPFS节点 (可选，用于离线消息)

### 快速开始

## API示例

### TypeScript客户端

```typescript
import { SignalKeysClient } from './SignalKeysClient';

async function main() {
  const client = await SignalKeysClient.connect();
  
  // 注册身份
  const account = client.createAccountFromUri('//Alice');
  await client.registerIdentity(account, identityKey, signedPrekey, signature);
  
  // 添加一次性预密钥
  await client.addOneTimePrekeys(account, [prekey1, prekey2, ...]);
  
  // 获取对方密钥
  const bobKeys = await client.getIdentity(bobAddress);
  const prekey = await client.getOneTimePrekey(bobAddress);
  
  // 开始X3DH密钥交换...
}

main();
```

### Rust协议使用

```rust
use signal_protocol::{
    keys::IdentityKeyPair,
    session::Session,
    message::PlaintextContent,
};

// 生成身份
let alice = IdentityKeyPair::generate()?;

// 创建会话
let (mut session, envelope) = Session::initiate(&alice, &bob_bundle)?;

// 加密消息
let plaintext = PlaintextContent::new("Hello!".to_string());
let encrypted = session.encrypt(&plaintext.to_bytes()?)?;

// 解密消息
let decrypted = bob_session.decrypt(&encrypted)?;
```

## 安全考虑

1. **前向保密**: Double Ratchet确保即使密钥泄露，历史消息仍安全
2. **身份验证**: 链上注册的身份密钥防止中间人攻击
3. **元数据保护**: P2P直连减少服务器暴露
4. **离线安全**: IPFS加密存储，仅接收方可解密
